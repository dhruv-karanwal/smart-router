Algorithm Explanation Cards
Dijkstra Algorithm
O((V+E) log V)

Finds the shortest path from a source node to all other nodes in a weighted graph using a min-heap priority queue. Guarantees optimal result with non-negative edge weights.

Priority Queue Min-Heap Non-negative weights
Use Case: Ambulance dispatch via shortest road route
A* Search Algorithm
Heuristic-based

Uses a heuristic function f(n) = g(n) + h(n) to guide the search toward the goal. Explores significantly fewer nodes than Dijkstra on spatial graphs, making it ideal for city maps.

Heuristic Search Euclidean h(n) Goal-driven
Use Case: Fire truck GPS routing in real-time
Bellman-Ford Algorithm
O(V × E)

Finds shortest paths by iteratively relaxing all edges V-1 times. Can handle negative edge weights and detect negative-weight cycles, making it more versatile than Dijkstra.

Edge Relaxation Negative weights Cycle detection
Use Case: Route with dynamic road congestion penalties
Floyd-Warshall Algorithm
O(V³)

Computes all-pairs shortest paths simultaneously using dynamic programming. Builds a V×V distance matrix, providing shortest paths between every pair of nodes in the graph.

Dynamic Programming All-pairs paths V×V matrix
Use Case: Pre-computing city-wide emergency routing tables
About

System Overview

An academic implementation of real-world emergency response route optimization, built using fundamental graph theory principles.

Data Structures Used

  • Adjacency List (Graph Representation)
  • Min-Heap / Priority Queue (Dijkstra, A*)
  • V×V Distance Matrix (Floyd-Warshall)
  • Hash Map for node distances

Algorithms Implemented

  • Dijkstra's Shortest Path
  • A* Search with Euclidean Heuristic
  • Bellman-Ford Relaxation
  • Floyd-Warshall All-Pairs Shortest Path

Use Cases

  • Ambulance dispatch routing
  • Fire truck real-time navigation
  • Police emergency response
  • Traffic signal pre-emption
